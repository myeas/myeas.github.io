## 1.精品配件存货账账相符的定义和作用

精品配件存货账账相符，指的是精品配件在采购、销售、库存环节，业务系统中的数据与财务系统中的数据是同步发生，期初、当期发生、期末的余额，业务系统和财务系统金额相等。

通过精品配件存货账账相符的核对，主要作用：

- 规范业务流程
- 确保业务账和财务账一致
- 提高数据准确性及使用价值
- 防控经营风险
- 促进业务顺畅流转
- 提高预算准确性

精品配件存货对账的主要内容：

- 采购环节：核对精品配件的采购入库单、应付单、入库成本调整单的单据和凭证是否一致；
- 采购环节：精品配件的采购入库与应付单的相互核销关系，应付单的未核销金额与采购入库单的未核销的差额，该差额与材料采购科目的余额是否一致；
- 库存环节：核对所有入库业务（包含采购入库、其它入库、入库成本调整单等）的单据和凭证是否一致。
- 库存环节：核对所有出库业务（包含销售出库、其他出库、领料出库、出库成本调整单等）的单据和凭证是否一致。
- 销售或维修环节：核对因销售或维修发生的精品配件出库的单据和凭证是否一致。
- 销售或维修环节：核对因销售或维修确认收入时，其成本结转的单据和凭证是否一致。

## 2.采购业务发生过程和对账主要内容

在实际业务发生过程中，物和票并不一定是同步到达发生的，因此为准确记录不同步的业务和财务发生，需要以应付暂付做为中转科目，在本项目上，以科目`材料采购-精品`、`材料采购-配件`（本项目充当了应付暂估科目）为承载，记录采购的精品配件入库、发票的记账过程、转移变化过程。

**借方，表示采购的精品配件已到采购发票；贷方，表示采购的精品配件已到货入库**。

`材料采购-精品`、`材料采购-配件`不得有手工凭证记载业务发生。

`材料采购-精品`、`材料采购-配件`以及整车业务的`材料采购-整车`（本章节附带提下）均需要关联辅助账类型`供应商`，在期末，财务根据这三个科目+供应商，来重分类，重分类后的借方余额计入存货，贷方余额计入应付账款。

### 2.1 采购业务发生过程

原则上，采购入库单、应付单**每行**记录的物料编码和名称、数量、价格、金额类数据，应当与供应商提供的原始单据、发票保持一致，不得张冠李戴，不得对不清扯不明。

#### 2.1.1 采购收到实物--采购入库单

当向供应商采购精品配件，收到实物时：

业务环节：

- 以`采购入库单`为承载记录发生过程。

财务记账环节：

- 以科目`材料采购-精品`，`材料采购-配件`贷方记录采购暂估过程（属于采购环节），

- 以科目`库存商品-精品`、`库存商品-配件`借方记录采购入库过程（属于库存环节）。

::: note 凭证参考--采购入库单

**采购入库生成凭证**时，不管采购入库，还是采购退货，其凭证格式为：

- 借：`1001 物料科目`，如：库存商品-精品/配件，金额取对应采购入库单上的实际成本
- 贷：`1004 材料采购科目`，如： **`材料采购-精品/配件，金额取对应采购入库单上的实际成本`**
- 采购入库单的业务日期与凭证的记账日期必须在同一期间
- 注意，在下述库存环节也讲到了采购入库单生成凭证，这个可不是重复生成凭证，采购环节侧重描述贷方材料采购科目，库存环节侧重描述借方库存商品科目

:::
::: tip 实际成本来源

- 依据供应商提供的报价，`去除增值税`后的不含税单价为`单位实际成本`，乘以数量后为`实际成本`。
- 特少情况下，供应商如无法提供可以抵扣进项税的发票，此时虽然提供的发票含税，但因不能抵扣，因此在系统中编制单据时，税率应该填0，此时也即不含税单价等于含税单价。一般应当要求供应商出具增值税专用发票，否则我们将损失此部分进项税额，无形中增加采购成本。
- 采购退货，除非经上级管理部门同意/门店负责人同意，不得折价退还。采购退货的同物料且同数量的实际成本，应当与正数的采购入库单的实际成本保持正负相反。即比如原来是采购A，1个10元，退货时，应当是A，-1个-10元，不应该不是10的值。

:::

::: danger 特别提醒--第一次提醒

- 禁止虚做采购入库单。

- 供应商赠送的或应当为0成本的，禁止通过采购入库流程完成入库，更不得虚指定所谓的0.01之类的单位成本。供应商赠送的，应当通过其他入库单，事务类型为`0921 零成本其他入库-赠送（库存类型为其它）`，实际成本为0的方式进行入库。

- 出库时，以销售出库单出库时，事务类型为：`0940 零成本销售出库（库存类型为其它）`。

  出库时，以其它出库单出库时，事务类型为：`0951 零成本其他出库（库存类型为其它）`。

- 上述这种虚采购入库，虚指定成本，首先会导致日常成本核算出错（因为出库成本是按加权核算得出），其次会导致可能出到最后，因为没有成本了，做普通出库时，会提示成本为0无法审核出库。仓库人员又想尽一切办法出库，接着就错上加错，以`0947 普通销售出库（零成本）（逐步禁用请不要再选择）`的事务类型出库。

- 所有事务类型含`（逐步禁用请不要再选择）`字样的，不得再用于日常业务。只允许调整以前遗留的错误。调整完毕后，后续任何业务均不得再使用。

- 如果不遵循上述要求，将导致无法出库、出库成本出错、业务和财务对账不平、几个库存报表之间（如库存类型收发汇总表及明细表、普通物料收发事务汇总表及明细、存货收发存汇总表及明细、库存账龄分析表等）不一致等一系列问题。

:::

#### 2.1.2 采购收到发票--应付单

当向供应商采购精品配件，收到发票时：

业务环节：

- 以`应付单`为承载记录发生过程。

财务记账环节：

- 以科目`材料采购-精品`，`材料采购-配件`借方记录采购发生过程，

- 以科目`应交税费-应交增值税-进项税额`借方记录发票的税额，

- 以科目`预付账款-预付精品配件款`贷方记录发票的含税金额，反映应付供应商的金额。
- 在实际操作过程中，可能因为四舍五入的原因，税额会与发票有微差，请在确保应付金额（含税金额）与厂家发票一致的前提下，可以适当调整，一般找金额较大的行，修改税额，金额会自动修改，确保每一行和合计行的应付金额、税额、金额，均与厂家发票一致（因前述微差调整的，视同一致）。

::: note 凭证参考--应付单

**应付单生成凭证**时，不管是正数发票，还是负数发票，其凭证格式为：

- 借：`1004 材料采购科目`，如：**`材料采购-精品/配件，金额取对应应付单上的金额`**
- 借：`1008 应交进项科目`，如：应付税费-应付增值税-进项税额，金额取对应应付单上的税额（如负数计入贷方正数，请依财务核算办法的科目要求和方向要求）
- 贷：`1203 物料预付科目`，如：预付账款-预付精品配件款，金额对应应付单上的应付金额
- 应付单的单据日期、业务日期与凭证的记账日期必须在同一期间

:::

::: tip 应付单金额决定采购的实际成本

- 依据供应商提供的发票，据实逐笔逐行核对，确保系统中的应付单，与实际采购发票一致。
- 金额，即不含税金额，是与采购入库核销的基准，据此决定采购的实际成本。
- 采购退货，除非经上级管理部门同意/门店负责人同意，不得折价退还。采购退货应付单的同物料且同数量的应付金额、金额、税额，应当与正数应付单的应付金额、金额、税额保持正负相反。即比如原来是采购A，1个，应付金额11.3元、金额10元、税额1.3元，退货时，应当是A，-1个，应付金额-11.3元、金额-10元、税额-1.3元。不应该是其他值。

:::

#### 2.1.3 采购入库与采购发票勾稽核销--采购发票核销记录和成本调整单（业务类型为入库）

当同一精品配件的实物和发票业务完成后，需要标记相互的匹配关系，在系统中以`核销`为承载，单据上以`已核销金额`/`已核销数量`、`未核销金额`/`未核销数量`记录发生过程，以核销记录承载核销结果。

**采购入库单**：

- 采购入库单的`已核销金额/已核销数量`，表示该采购入库单确认了多少金额和数量的采购发票，与应付单核销了多少。
- 或收到了实物又因各种原因退还给供应商，而供应商对正数采购、负数退货均不再开具采购发票，门店可自行确认采购入库和采购退货的匹配关系，对正负采购入库单进行手工核销。
- 特别提醒：手工正负核销时，必须确保同物料正负数量的实际成本正负相等。比如正数A，1个，实际成本100元，则负数A，-1个，实际成本为-100元。否则不得手工正负核销。

- 采购入库单的`未核销金额/未核销数量`，表示精品配件已采购入库但还未到采购发票的金额和数量。

**应付单**：

- 应付单的`已核销金额/已核销数量`，表示该应付单确认了多少金额和数量的采购入库单，与采购入库单（并入库成本调整单）核销了多少。
- 一般不做正负应付单之间的核销操作。

- 应付单的`未核销金额/未核销数量`，表示精品配件已收到采购发票但还实物还没有入库的数量和金额。

**入库成本调整单**：

**当采购入库单与应付单核销有差额时，核销处理过程中系统会`自动生成业务类型为入库的成本调整单`（简称入库成本调整单）**。

::: note 凭证参考--入库成本调整单

**入库成本调整单生成凭证**时，不管正数差额还是负数差额，其凭证格式为：

- 借：`1001 物料科目`，如：库存商品-精品/配件，金额对应入库成本调整单上的调整金额
- 贷：`1004 材料采购科目`，如：**`材料采购-精品/配件，金额对应入库成本调整单上的调整金额`**
- 核销记录，不需要生成凭证
- 入库成本调整单的业务日期与凭证的记账日期必须在同一期间
- 注意，在下述库存环节也讲到了入库成本调整单生成凭证，这个可不是重复生成凭证，采购环节侧重描述贷方材料采购科目，库存环节侧重描述借方库存商品科目

:::

::: danger 特别提醒

- 在核销操作时，如是采购入库单的正数与负数之间进行核销，务必确保本次核销数量、本次核销金额要正负匹配，即同一物料的正的本次核销金额+负的本次核销金额应等于0、同一物料的正的本次核销数量+负的本次核销数量应等于0。
- 精品配件的**应付单，必须由采购业务流转的单据关联生成**，由采购订单或采购入库单关联生成，即应付单分录的核心单据类型为`采购订单`。实际应为采购业务的应付单，但在系统中却是手工新增的（即应付单分录的核心单据类型为空），为不规范业务，会导致采购环节的对账不平。
- **入库成本调整单，必须由核销处理时，由系统自动生成**，`不得手工新增`。采购入库单的`实际成本`字段与对应应付单的`金额`字段有差异时，触发生成入库成本调整单。
- 材料采购、库存商品的凭证，必须由单据生成，禁止手工编制凭证。
- <u>精品配件的采购，**一般是先入库后到票**，因此当采购业务的应付单有未核销数量、未核销金额时，即**先到票还未入库，应当予以注意**，并确认实际是否有已到票未入库的业务，如实际没有但系统有未核销数量、未核销金额，则肯定有某些操作是错误的，请自行核查，并尽快修正</u>。

:::

### 2.2 采购业务对账主要内容

**日常发生**：

通过上述发生过程，我们可以很容易得到如下恒等关系，本部也是基于这些业务恒等逻辑，要求各店进行业务数据和财务数据的核对。

::: note 采购对账--日常对账

- **科目`材料采购-精品/配件`的`贷方发生额`  =  同期`采购入库单`的`实际成本`  +  同期`入库成本调整单`的`调整金额`**
- **科目`材料采购-精品/配件`的`借方发生额`  =  同期`应付单`的`金额`**
- 上述恒等式，可以精确到具体的供应商
- 上述恒等式，请自行筛查科目余额表、采购入库单、入库成本调整单、应付单即可，不单独提供报表

:::

同时，采购入库单上的已核销数量/金额、未核销数量/金额，应付单上的已核销数量/金额、未核销数量/金额，会随着业务的流转发生变化，同一笔业务最终完成后，采购入库单上的已核销数量会等于数量、已核销金额会等于入库成本，未核销数量和未核销金额为0。应付单上的已核销数量会等于数量、已核销金额会等于金额，未核销数量和未核销金额为0。

**期末余额**：

结合业务发生过程、核销记录，我们可以还原出月末时点的未核销数量、未核销金额，因此还可以得到如下恒等关系，本部也是基于这些业务恒等逻辑，要求各店进行业务数据和财务数据的核对。

::: note 采购对账--期末对账

- **科目`材料采购`的`期末余额`  =  借（`应付单未核销金额C`  +  `应付历史核销金额D`）  -  贷（`采购入库单未核销金额A`  +  `采购入库历史核销金额B`）**
- 上述恒等式，可以精确到具体的供应商
- 上述恒等式，可以通过《精品配件存货对账表》查询，如有差异，则为异常，需要及时核查并纠正

:::

### 2.3 采购业务对账查询参考指引

以下查询指引，以截至2022年02月28日的数据为例。

每月存货核算模块结账前，必须查询《精品配件存货对账表》，查看材料采购是否有差异，如有差异，则需要及时核查并纠正错误。

- 核查《精品配件存货对账表》是否有差异，如没有差异，则正常，如有差异，则需要核查本期发生、本期核销记录。
- 核查材料采购本期借方发生，是否有差异。
- 核查材料采购本期贷方发生，是否有差异。
- 核查精品配件核销是否有异常核销操作（常见的为正负入库核销时，核销金额不相等）。

#### 2.3.1 材料采购期末余额核对方法

**对账报表：精品配件存货对账表**

- **对账公式**：`科目材料采购的期末余额  =  借（应付单未核销金额C  +  应付历史核销金额D）  -  贷（采购入库单未核销金额A  +  采购入库历史核销金额B）`
- **报表路径**：〖财务会计〗-〖存货核算〗-〖期末处理〗-〖精品配件存货对账表〗

#### 2.3.4 精品配件异常核销<Badge text="常被忽略" type="warning" vertical="top"/>

精品配件的异常核销，通常指采购入库单正负核销时，其核销数量正负不相等或核销金额不相等。

当月末核查《精品配件存货对账表》时，科目借方余额与业务对得上，科目贷方余额与业务对得上，期初也是平的，但期末就是不平，就要考虑可能的原因是本项异常核销的存在。

如果核销期间是本期，需要门店自行反核销，不能再手工正负金额不等的核销了，而应该通过补推正负应付单的方式处理，并确保被推的虚做的正负应付单的应付金额正负相等、金额正负相等、税额正负相等，这正负应付单合并生成凭证后，借贷都是0，不影响财务发生、余额、对账等。此时系统会自动生成入库成本调整单，对入库成本调整单生成凭证，就可以了。

如果核销期间是历史期间，则需要反馈给本部，由本部进行反核销，然后按上一段描述的方法补正负应付单进行处理。

### 2.4 采购业务对账出现差异常见的原因

导致采购业务对账出现差异常见的原因，主要有：

#### 2.4.1 物料未绑定记账分类

这个问题，一般常见于新增的物料。

当在单据生成凭证时，无法带出科目时，一定要去检查对应的物料是否绑定记账分类。

#### 2.4.2 记账要素未关联准确的科目

这个问题，常发生刚使用该业务模块的门店。

要先设立记账分类，再在记账分类中对记账要素关联科目。

记账分类路径：〖企业建模〗-〖辅助数据〗-〖财务会计数据〗-〖记账分类〗

`记账分类类型`，请务必切换到`物料`，再新增记账分类，一般共三个：`整车`，`精品`，`配件`。

记账要素`1001 物料科目`，需要按照该物料是精品还是配件，关联到`1405.02.02 库存商品-精品`或`1405.02.03 库存商品-配件`；

记账要素`1004 材料采购科目`，需要按照该物料是精品还是配件，关联到`1401.98 材料采购-精品`或`1401.99 材料采购-配件`；

记账要素`1203 物料预付科目`，关联到`1123.54.02 预付精品配件款`（个别门店未设立三级科目，则关联为`1123.54 预付商品款`）。

#### 2.4.3 采购入库单生成凭证时与要求的格式不相符或金额不相等

凭证样式，请参照前述记账规范，按规范对单据生成凭证。

以下行为会导致错误：

- 生成凭证后，手工切换分录的方向；
- 生成凭证后，手工修改生成的金额；
- 生成凭证后，没有带出科目时，没有及时对物料绑定记账分类，而是直接在凭证上手工选科目；
- 生成凭证后，手工增加、修改或删除了辅助账；
- 生成凭证后，手工修改了带出来的科目。

#### 2.4.4 入库成本调整单生成凭证时与要求的格式不相符或金额不相等

凭证样式，请参照前述记账规范，按规范对单据生成凭证。

以下行为会导致错误：

- 生成凭证后，手工切换分录的方向；
- 生成凭证后，手工修改生成的金额；
- 生成凭证后，没有带出科目时，没有及时对物料绑定记账分类，而是直接在凭证上手工选科目；
- 生成凭证后，手工增加、修改或删除了辅助账；
- 生成凭证后，手工修改了带出来的科目。

#### 2.4.5 存在手工的入库成本调整单

禁止手工做入库成本调整单，如有，请删除。必须由系统根据采购核销逻辑自动生成。

#### 2.4.6 应付单生成凭证时与要求的格式不相符或金额不相等

凭证样式，请参照前述记账规范，按规范对单据生成凭证。

以下行为会导致错误：

- 生成凭证后，手工切换分录的方向；
- 生成凭证后，手工修改生成的金额；
- 生成凭证后，没有带出科目时，没有及时对物料绑定记账分类，而是直接在凭证上手工选科目；
- 生成凭证后，手工增加、修改或删除了辅助账；
- 生成凭证后，手工修改了带出来的科目。

#### 2.4.7 存在手工凭证或其他非采购业务的单据生成凭证时记录到了材料采购

禁止手工编制，包含材料采购、库存商品、预付账款科目的凭证。

#### 2.4.8 精品配件的正负采购入库单核销时，核销金额正负不相等

**精品配件的正负采购入库单手工核销时，请务必确保正负数量、正负金额相等。否则将导致精品配件存货对账不平**。

#### 2.4.9 采购入库单、入库成本调整单、应付单的业务归属期间与对应凭证的归属期间不一致

在生成凭证前，筛选并选中时，不要跨月筛选。生成凭证后，不要改凭证上的记账日期。

如果是单据上错了，应该更改原始单据上的日期（业务日期、单据日期、调整日期）。

生成凭证后，凭证上的记账日期归属期间，必须和上游单据的日期（业务日期、单据日期、调整日期）归属期间在同一期间。

#### 2.4.10 采购时，错把数量与金额弄反

实际上，这个属于精品配件存货账实问题，必然导致系统入库的数量与实际不符，按动态盘点要求，应当及时发现，发现后及时修正问题。删除原单据再被做正确单据，或不动原单据再冲掉原单据，再冲增正确单据。

### 2.5 精品配件采购业务可能出现的问题

通过存货账账相符，确保业务账和财务账一致后，我们再核查实际的采购未核销、应付单未核销时，可能就会发现一些问题，包含但不限于如下问题：

::: warning 常见问题

- 长时间采购入库未核销，如果确实压供应商，也与实际相符，可以理解，但如果不是压供应商，或与实际不相符，则应核查；
- 采购入库未核销的明细里，未核销数量为0，但未核销金额不为0，这一般是手工错误核销导致的；
- **`如果存在长时间的负数采购入库未核销，需尽快与供应商索要负数发票，否则将给企业带来损失`**；
- **`一般来说，精品配件都是先到货后到票，如果应付单有未核销金额，则业务人员和财务人员均应该即刻意识到，这是错误的结果，则应核查`**；
- 在日常采购过程中，存在不严谨的入库或应付单的推单操作，如物料、数量、金额与实际不相符，强烈要求日常制单时加强与原始单据的核对；
- 如果按月或按一定周期结算的，一般都是以销售及内部耗用的数量为与供应商的结算依据，则还原后的期末未核销数量与期末库存数量，原则要一致，即在库的数量为还没有与供应商结算的数量，如果不一致，则应核查；
- 可能存在一个订单重复推采购入库单、一个采购入库单推重复的应付单的问题，强烈建议日常制单及时审核，在审核前可以上查到上游单据，再在上游单据下查，日常制单时可看看下游单据是不是有重复的单据；
- 可能存在采购退货时，或因错误入库通过采购退货重新入库时，入库单与应付单的匹配关系不正确，比如原入库、新退货、新入库，此时应付单应该推在新入库单下，不应该在原入库单下推应付单，如果确发生此操作，则需要对新退货、新入库分别补推应付单。

:::

欢迎门店反馈，日常业务过程还会遇到的问题。

### 2.6 采购业务对账出现差异的核查思路

核查思路：

1. 查询《精品配件存货对账表》，每月存货核算模块结账前，必须核对，确保无差异；
2. 按上述日常业务发生与日常财务发生之间是否有差异；
3. 核查手工采购核销是否有正负核销金额不一致的情况。

处理思路：

1. 当日常发生不一致时，请结合实际原始单据、采购发票等凭据，并结合上述差异原因，判断是业务单据错误，还是凭证错误。如果是当期，则删除错误单据，重新编制；如果是跨期，则按照调整总原则`原来如何错误的，则先以同样错误的方式并以相反金额冲掉原来的单据，再按正确的操作补调单据`，冲销并调整单据和凭证。
2. 如果是手工正负核销异常，联系本部进行反核销，再重新虚推正负应付单（应付金额、金额、税额，均正负一致），让差异通过自动产生的入库成本调整单的方式调整。

## 3 库存业务发生过程和对账主要内容

本小节描述精品配件发生增加减少的变化过程，涉及到采购环节的业务（比如采购订单、应付单、采购核销等）请参见上一节内容；涉及到销售环节的业务（比如精品销售单、维修工单、应收、销售核销、结转成本等）请参见下一节内容。

精品配件库存业务发生增加减少的变化过程：

- 在系统中以**库存单据**为承载
- 以精品配件实物的入库和出库的时点为发生时点
- 据实有序记录每一笔库存变化过程

::: details 库存单据的主要字段

- **业务日期**：精品配件等实物进入仓库或从仓库发出的时点；

- **供应商/客户**：精品配件等实物转移的对象；供应商，谁提供；客户：给谁；

- **事务类型**（库存状态、库存类型、更新类型）：核心字段。事务类型是融合库存状态、库存类型、更新类型等属性而成的一组描述业务单据业务属性的关键特征信息，决定增加/减少库存（数量/金额）、参与/不参与核算、核销/不核销、对账/不参与对账、归属的收发类型等；

- **物料**：以编码为唯一识别信息，全系统唯一，即使实物为同一物，但当在系统中以不同编码记录时，则在系统中认定为不同的实物，互不相关；通过基本分类划分物料组别；

- **基本数量/数量**、**基本计量单位/计量单位**：系统支持丰富的计量单位设置及换算，基本资料中的基本计量单位、采购资料中的采购计量单位、库存资料中的库存计量单位、销售资料中的销售计量单位，各计量单位通过`多计量单位`设定换算关系，同时还支持辅助计量单位等。但**在本项目中，并没有那么复杂的业务诉求，因此要求各门店及本部维护物料的人员，在维护物料时，必须保持各资料页签中的计量单位保持一致，否则将在实际业务操作中带来操作困惑，显著降低工作效率、显著增加错误**。

  计量单位：数量组：以最小包装为计量单位，不可再拆分使用为基准，计量单位选择`G001 EA`，不具体到如件、瓶、个、块、卷、台、套、条、张、支等；其`计量精度`要求设置为`0`，即以整数入库或出库；

  以体积为计量的物料（如机油），且散装按需抽取使用的，以`SHENG 升`为计量单位；其计量精度按需设置，一般不超过3；

  虽然主要计量也为体积，但以整个最小包装的入出库的物料（如机油），不建议使用小桶、大桶等太笼统的名称，建议仍选择`G001 EA`为计量单位，并辅助在物料名称上标记规格（如1\*1L、1\*4L）；

- **仓库/库位**：物料存放的位置；要求所有仓库均设置库位管理；

- **实际成本/调整金额**：承载库存单据增加或减少的金额信息；

- **记账分类**：若干记账要素形成一组的总称，一般设置为`整车`、`精品`、`配件`，记账要素是财务核算记账的最小依据，比如`1001 物料科目`，当入库时，借方即取`1001 物料科目`关联的科目；

:::

### 3.1 库存单据种类

**入库单据及事务类型**：

正数：增加库存数量（及库存金额）；

负数：减少库存数量（及库存金额）。

::: danger 特别提醒--第二次提醒

- 禁止虚做采购入库单。

- 供应商赠送的或应当为0成本的，禁止通过采购入库流程完成入库，更不得虚指定所谓的0.01之类的单位成本。供应商赠送的，应当通过其他入库单，事务类型为`0921 零成本其他入库-赠送（库存类型为其它）`，实际成本为0的方式进行入库。

- 出库时，以销售出库单出库时，事务类型为：`0940 零成本销售出库（库存类型为其它）`。

  出库时，以其它出库单出库时，事务类型为：`0951 零成本其他出库（库存类型为其它）`。

- 上述这种虚采购入库，虚指定成本，首先会导致日常成本核算出错（因为出库成本是按加权核算得出），其次会导致可能出到最后，因为没有成本了，做普通出库时，会提示成本为0无法审核出库。仓库人员又想尽一切办法出库，接着就错上加错，以`0947 普通销售出库（零成本）（逐步禁用请不要再选择）`的事务类型出库。

- 所有事务类型含`（逐步禁用请不要再选择）`字样的，不得再用于日常业务。只允许调整以前遗留的错误。调整完毕后，后续任何业务均不得再使用。

- 如果不遵循上述要求，将导致无法出库、出库成本出错、业务和财务对账不平、几个库存报表之间（如库存类型收发汇总表及明细表、普通物料收发事务汇总表及明细、存货收发存汇总表及明细、库存账龄分析表等）不一致等一系列问题。

:::

::: details 采购入库单（不含退字样为正数增加库存数量/金额，含退字样为负数减少库存数量/金额）

- **采购入库单**：适用于向供应商采购，依据采购发票需要向供应商支付对价的业务

  事务类型：`001 普通采购/委外入库`，库存类型为普通，收到供应商实物，**增加**本组织库存数量和金额；需要向供应商支付对价；

  事务类型：`002 普通采购/委外退库`，库存类型为普通，退还供应商实物，**减少**本组织库存数量和金额；需要向供应商支付对价（负数）；一般适用于冲减「001 普通采购/委外入库」的业务；

:::

::: details 其他入库单（不含退字样为正数增加库存数量/金额，含退字样为负数减少库存数量/金额）

- **其他入库单**：适用于供应商赠送无需支付对价、实物盘盈等业务

  事务类型：`032 无来源手工入库`，库存类型为普通，**增加**本组织库存数量和金额；无需向任何单位支付对价；非本部指导下不得使用；

  事务类型：`0321 无来源手工退库`，库存类型为普通，**减少**本组织库存数量和金额；无需向任何单位支付对价；一般适用于冲减「032 无来源手工入库」的业务；

  事务类型：`0920 零成本其他入库-盘盈（库存类型为其它）`，库存类型为其它，**增加**本组织库存数量、不增加金额（即实际成本要求为0）；

  事务类型：`0921 零成本其他入库-赠送（库存类型为其它）`，库存类型为其它，**增加**本组织库存数量、不增加金额（即实际成本要求为0）；

  事务类型：`0922 零成本其他入退库（库存类型为其它）`，库存类型为其它，**减少**本组织库存数量、不增加金额（即实际成本要求为0）；一般适用于冲减「0920 零成本其他入库-盘盈（库存类型为其它）」、「0921 零成本其他入库-赠送（库存类型为其它）」的业务；

  事务类型：`0971 其他入库（索赔旧件-库存类型为其它）`，库存类型为其它，**增加**本组织库存数量、不增加金额（即实际成本要求为0）；一般适用于记录保修作业过程中换下的零件，且后续需要向厂商返还或经厂家同意后销毁；

  事务类型：`0972 其他入库（保险旧件-库存类型为其它）`，库存类型为其它，**增加**本组织库存数量、不增加金额（即实际成本要求为0）；一般适用于记录事故维修作业过程中换下的零件，且后续需要向保险公司返还或经保险公司同意后销毁；

  事务类型：`0923 赠品其他入库（库存类型为赠品）`，库存类型为赠品，**不再使用**；

  事务类型：`0924 赠品其他入退库（库存类型为赠品）`，库存类型为赠品，**不再使用**；冲减「0923 赠品其他入库（库存类型为赠品）」；

  事务类型：`0927 其他入库（零成本）（逐步禁用请不要再选择）`，库存类型为普通，容易导致成本核算出现错误，**禁止使用**，如历史还有此类库存，需尽快处理；

  事务类型：`0928 其他入退库（零成本）（逐步禁用请不要再选择）`，库存类型为普通，容易导致成本核算出现错误，**禁止使用**，冲减「0927 其他入库（零成本）（逐步禁用请不要再选择）」；

- 其他入库中的索赔旧件、理赔旧件的入库，在系统中主要是记录下损坏件/旧件的数量管理过程，其没有实际价值，更不能用于日常销售或维修业务中。

:::

::: details 入库成本调整单（正数增加库存金额、负数减少库存金额）

- **入库成本调整单**（成本调整单，业务类型为入库，简称入库成本调整单）

  入库成本调整单，**禁止手工新增**，必须经采购环节的核销操作由系统自动产生，特征：入库成本调整单上的`创建类型`为`采购-自动生成`。

  因采购环节的业务，库存类型都是普通、库存状态都是可用，所以自动生成的入库成本调整单，其库存类型同样为`普通`、库存状态同样为`可用`。

  业务类型：`入库`。

:::

::: details 调拨入库（增加库存数量/金额，必须同时有调拨出库）

- **调拨入库单**：适用于本门店内部，精品配件从一仓库及（或库位）调拨到另一仓库及（或库位）的业务；**调拨业务必须同时有调拨出库和调拨入库**。

  事务类型：`028 普通调拨入库`，适用于即时库存的库存类型为`普通`的调拨入库业务；同步的调拨出库，单据上的事务类型必须为`027 普通调拨出库`；先调拨出库再调拨入库；

  事务类型：`4333-2 调拨入库（库存类型为其它）`，适用于即时库存的库存类型为`其它`的调拨入库业务；同步的调拨出库单，单据上的事务类型必须为`4333-2 调拨入库（库存类型为其它）`；

  调拨入库单和调拨出库单，均必须由同一上游库存调拨单生成，且上游库存调拨单的`业务类型`必须为`331 跨仓库调拨（成本价）`；

### 3.2 库存类型/事务类型

通过上述库存单据种类的区分，我们可以发现，事务类型根据库存类型可以划分为四组：

- **普通**：库存类型为普通 ，最常见的业务，**核算数量和成本**，即选用了这类事务类型的单据，其单据上的`实际成本`或`调整金额`**必须有值**；所有不含`库存类型为其它`、`库存类型为赠品`，`逐步禁用`字样的事务类型，其库存类型均为普通；
- **其它**，库存类型为其它，较常见的业务，**核算数量，不核算成本**，即选用了这类事务类型的单据，其单据上的`实际成本`或`调整金额`**不得有值**；
- **赠品**，库存类型为赠品，已禁止使用，作用实际上和`其它`一样，只不过因为其有赠品字样，所以不再使用；
- **逐步禁用**，库存类型为普通，已严禁使用，使用这些事务类型，将`导致成本出现错误`。<u>**如历史已使用过，且当前有库存数量，需要尽快调整。在调整前，先进行核查，再联系本部予以临时启用这些事务类型，并按本部指导要求调整库存数量和库存金额**</u>。

::: danger 特别强调

- 库存数量和金额要发生入库或出库，**必须在同组内发生**，否则将导致数量、成本出错，或者提示库存不足；

- 普通组的单据，必须有数量和实际成本（调整金额），否则是错误的；

  其它组、赠品组、逐步禁用组，必须有数量，且必须实际成本（调整金额）为0，否则是错误的；

- 库存数量的增加或减少，是以 **`物料`+`批次`+`计量单位`+`基本计量单位`+`辅助计量单位`+`仓库`+`库位`+`库存类型`+`库存状态`+`库存组织`** 的组合维度来增加或减少的，因此当组合维度不一致时，增加时会形成新的组合数据，减少时就可能导致无法出库，提示库存不足。库存金额，同理。
- **最常见的出错，大多是因为`忽略三组计量单位、库存类型`**，其他维度比较好理解（如物料、仓库、库位）或基本不用（批次）或所有均一致（如库存状态）不太会出错。

:::

:::: details 事务类型及库存类型分组（必须同组别入库和出库）

所有入库和出库必须在同组别内进行，否则将导致但不限于如下问题：

- 无法出库（如入库为组别二，出库想出在组别一 等等）
- 库存混乱（如出库为组别二但退回时却在组别一，或如入库在组别一但退回时却在组别二 等等）
- 成本核算错误（如入库在组别四但出库在组别一，或如入库在组别一但出库在组别四 等等）

::: warning 特别提醒--反复提醒

- 禁止虚做采购入库单。

- 供应商赠送的或应当为0成本的，禁止通过采购入库流程完成入库，更不得虚指定所谓的0.01之类的单位成本。供应商赠送的，应当通过其他入库单，事务类型为`0921 零成本其他入库-赠送（库存类型为其它）`，实际成本为0的方式进行入库。

- 出库时，以销售出库单出库时，事务类型为：`0940 零成本销售出库（库存类型为其它）`。

  出库时，以其它出库单出库时，事务类型为：`0951 零成本其他出库（库存类型为其它）`。

- 上述这种虚采购入库，虚指定成本，首先会导致日常成本核算出错（因为出库成本是按加权核算得出），其次会导致可能出到最后，因为没有成本了，做普通出库时，会提示成本为0无法审核出库。仓库人员又想尽一切办法出库，接着就错上加错，以`0947 普通销售出库（零成本）（逐步禁用请不要再选择）`的事务类型出库。

- <u>所有事务类型含`（逐步禁用请不要再选择）`字样的，不得再用于日常业务。只允许调整以前遗留的错误。调整完毕后，后续任何业务均不得再使用。</u>

- 如果不遵循上述要求，将导致无法出库、出库成本出错、业务和财务对账不平、几个库存报表之间（如库存类型收发汇总表及明细表、普通物料收发事务汇总表及明细、存货收发存汇总表及明细、库存账龄分析表等）不一致等一系列问题。

### 3.3 库存业务发生过程

::: danger 特别提醒--第三次提醒

- 禁止虚做采购入库单。

- 供应商赠送的或应当为0成本的，禁止通过采购入库流程完成入库，更不得虚指定所谓的0.01之类的单位成本。供应商赠送的，应当通过其他入库单，事务类型为`0921 零成本其他入库-赠送（库存类型为其它）`，实际成本为0的方式进行入库。

- 出库时，以销售出库单出库时，事务类型为：`0940 零成本销售出库（库存类型为其它）`。

  出库时，以其它出库单出库时，事务类型为：`0951 零成本其他出库（库存类型为其它）`。

- 上述这种虚采购入库，虚指定成本，首先会导致日常成本核算出错（因为出库成本是按加权核算得出），其次会导致可能出到最后，因为没有成本了，做普通出库时，会提示成本为0无法审核出库。仓库人员又想尽一切办法出库，接着就错上加错，以`0947 普通销售出库（零成本）（逐步禁用请不要再选择）`的事务类型出库。

- 所有事务类型含`（逐步禁用请不要再选择）`字样的，不得再用于日常业务。只允许调整以前遗留的错误。调整完毕后，后续任何业务均不得再使用。

- 如果不遵循上述要求，将导致无法出库、出库成本出错、业务和财务对账不平、几个库存报表之间（如库存类型收发汇总表及明细表、普通物料收发事务汇总表及明细、存货收发存汇总表及明细、库存账龄分析表等）不一致等一系列问题。

:::

精品配件的库存实际业务发生过程中，以实物的转移时点为发生时点，在本项目上，业务上以`库存单据`为承载，财务上以科目`库存商品-精件`、`库存商品-配件`为承载，记录实物库存的转移变化过程。

借方，对应库存单据的入库单据，表示实物的收入业务（正数增加库存数量/金额，负数减少库存数量/金额）；

贷方，对应库存单据的出库单据，表示实物的发出业务（正数减少库存数量/金额，负数增加库存数量/金额）；

通过前述事务类型的描述，入库并不总表示增加，出库也并不总表示减少，为准确记录实际业务发生过程，反映同类型业务，系统中设立了负数事务类型，以包含`退`字样的事务类型名称为承载。

因此，

- 当采购入库，需要退还给供应商时，应当以采购退货的方式退还供应商，而不是销售出库或其他出库等；
- 当销售出库，需要从车间退还、从客户退还，应当以销售退货的方式退给仓库，而不是其他入库、采购入库等；
- 市场活动领用的物料，当活动结束后，多余的物料退还给仓库时，不应以其他入库单的方式入库，而应以领料出库单退库方式（原出库时以领料出库单）或其他出库单退库方式（原出库时以其他出库单）。
- 以此类推。



#### 3.3.1 采购收到实物-采购入库单

当向供应商采购精品配件，收到实物时，业务上以`采购入库单`为承载记录发生过程，财务记账时以科目`材料采购-精品`，`材料采购-配件`贷方记录采购暂估过程（属于采购环节），以科目`库存商品-精品`、`库存商品-配件`借方记录采购入库过程（属于库存环节）。

::: note 凭证参考-采购入库单

**采购入库生成凭证**时，不管采购入库，还是采购退货，由采购入库单【生成凭证】，其凭证格式为：

- 借：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应采购入库单上的实际成本`**
- 贷：`1004 材料采购科目`，如：材料采购-精品/配件，金额取对应采购入库单上的实际成本
- 采购入库单的业务日期与凭证的记账日期必须在同一期间
- 注意，在前述采购环节也讲到了采购入库单生成凭证，这个可不是重复生成凭证，采购环节侧重描述贷方材料采购科目，库存环节侧重描述借方库存商品科目

:::

请注意：一个单据，只能生成一个凭证；若干单据可以合并生成一个凭证，不能一个单据生成若干凭证。

#### 3.3.2 收到供应商赠送/内部盘盈等-其他入库单

其他入库的主要业务场景：

- 当供应商因其销售政策向本组织赠送精品配件时。一般实际成本为0。
- 当内部定期盘点，发生盘盈时。注意所有的盘盈，必须先核对原因，并经部门经理、财务、店总审批，确认为盘盈，方可以其他入库单记录，避免因其他错误的操作导致的非真正盘盈（如实际为采购入库的，没及时采购入库的等等各种原因）。实际成本是否为0，请结合实际业务管理和本部财务规范。一般参考为0。
- 当维修作业，应当给客户但客户不要，或因单个最小包装出库但实际未完全用完的（如更换机油多余的）。实际成本是否为0，请结合实际业务管理和本部财务规范。一般参考为0。
- 因索赔维修、理赔维修，换下的旧件，为便于记录旧件的转移过程，宜以其他入库单记录。实际成本为0。

::: note 凭证参考-其他入库

**其他入库生成凭证**时，不管正数入库，还是负数入库，由其他入库单【生成凭证】，其凭证格式为：

- 借：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应其他入库单上的实际成本`**
- 贷：**需手工选择**，如：营业外收入/（或借方冲减费用、成本等），该科目具体为哪个科目，根据实际承载的业务、本部财务核算要求、国家财税政策等，金额取对应其他入库单上的实际成本
- 其他入库单的业务日期与凭证的记账日期必须在同一期间

:::

请注意：一个单据，只能生成一个凭证；若干单据可以合并生成一个凭证，不能一个单据生成若干凭证。

#### 3.3.3 采购核销差异-入库成本调整单（业务类型为入库）

当同一精品配件的实物和发票业务完成后，需要标记相互的匹配关系，在系统中以`核销`为承载，单据上以`已核销金额`/`已核销数量`、`未核销金额`/`未核销数量`记录发生过程，以核销记录承载核销结果。当采购入库单与应付单核销有差额时，核销处理过程中系统会`自动生成业务类型为入库的成本调整单`（简称入库成本调整单）。

::: note 凭证参考-入库成本调整单

**入库成本调整单生成凭证**时，不管正数差额还是负数差额，由入库成本调整单【生成凭证】，其凭证格式为：

- 借：`1001 物料科目`，如：**`库存商品-精品/配件，金额对应入库成本调整单上的调整金额`**
- 贷：`1004 材料采购科目`，如：材料采购-精品/配件，金额对应入库成本调整单上的调整金额
- 核销记录，不需要生成凭证
- 入库成本调整单的业务日期与凭证的记账日期必须在同一期间
- 注意，在上述采购环节也讲到了入库成本调整单生成凭证，这个可不是重复生成凭证，采购环节侧重描述贷方材料采购科目，库存环节侧重描述借方库存商品科目

:::

::: warning 提醒

- 不得有任何手工新增的入库成本调整单。

:::

请注意：一个单据，只能生成一个凭证；若干单据可以合并生成一个凭证，不能一个单据生成若干凭证。

#### 3.3.4 因销售或维修向客户交付实物-销售出库单

当客户进店购买精品配件的精品配件销售业务向客户交付实物，或车辆进厂维修需要领用配件交付实物，业务上以`销售出库单`为承载记录实物发生过程，财务记账时以科目`库存商品-精品`，`库存商品-配件`贷方记录实物变化过程。

同时，销售性质（含维修）的实物交付需要确认为成本，并需要业务完结后确认收入，而确认收入的时点与实物出库的时点，并不完全同步，因此为适应此业务特点，本项目财务记账时以科目`发出商品-精品配件销售`、`发出商品-维修`记录库存商品转入主营业务成本的中间过程。借方，表示实物已出库但未确认成本；贷方，表示已计入发出商品的，在确认收入后结转成本。（本段描述，也是下述销售发出/维修发出环节的重要基准）。

::: note 凭证参考--销售出库单（维修出库）

**销售出库单生成凭证**时，不区分是正数出库还是负数退货，但区分其来源，来源于维修工单的，由核心单据类型为`维修工单`的销售出库单【生成凭证】，其凭证格式为：

- 借：统一指定为`1406.01 发出商品-维修`，金额取对应销售出库单上的实际成本
- 贷：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应销售出库单上的实际成本`**
- 销售出库单的业务日期与凭证的记账日期必须在同一期间
- 注意，在下述销售或维修环节也讲到了销售出库单生成凭证，这个可不是重复生成凭证，库存环节侧重描述贷方库存商品科目，销售环节侧重描述发出商品科目

:::

::: note 凭证参考--销售出库单（精品配件销售出库）

**销售出库单生成凭证**时，不区分是正数出库还是负数退货，但区分其来源，来源于精品配件销售单的，由核心单据类型为`精品销售单`的销售出库单【生成凭证】，其凭证格式为：

- 借：统一指定为`1406.02 发出商品-精品配件销售`，金额取对应销售出库单上的实际成本
- 贷：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应销售出库单上的实际成本`**
- 销售出库单的业务日期与凭证的记账日期必须在同一期间
- 注意，在下述销售或维修环节也讲到了销售出库单生成凭证，这个可不是重复生成凭证，库存环节侧重描述贷方库存商品科目，销售环节侧重描述发出商品科目

:::

::: warning 实际成本来源

- 不管是维修出库，还是精品配件销售出库，甚至是其他出库、领料出库、调拨出库，其实际成本来源的方式是一样的。

- 核心逻辑：`根据物料的计价方法，依据单据的生效时间，有序核算成本`。

- 本项目，精品配件的计价方法均要求为`实时移动加权平均法`。其显著特征是，`单据审核时即核算出库成本`，如果事务类型为普通，但后台核算的成本小于等于0，在审核时即会报错，需立即通过《库存类型收发明细表》核查该物料的收发情况，以判断是哪里出错，纠正错误后再审核出库单。

- 月末还可以通过`材料出库核算`功能，重新对出库单的实际成本进行核算，核算前，要重算的单据不能生成凭证，否则将不会更新已生成凭证的各出库单。

- 按事务类型的组别进行成本核算，不同组别不相互影响成本。特例，实际上组别四（逐步禁用组）和组别一，其库存类型均为普通，实际上这两组会混和在一起核算成本，导致成本核算出错，因此严格禁止使用组别四的各事务类型，历史单据遗留的数据，需尽快处理。

- 库存类型为`普通`的（参与核算，组别一），必须`有实际成本`；

  库存类型为`其它`的，必须`实际成本为0`；

  库存类型为`赠品`的，必须`实际成本为0`；

  库存类型为`普通的（不参与核算，组别四，即事务类型名称含`逐步禁用请不要再选择`字样）`，必须`实际成本为0`，如不为0则是错误的结果。

  **<u>有组别四入库，就必须得为组别四出库。不得组别一入库而组别四出库，或组别四入库而组别一出库，如此则会导致成本核算出错</u>**。

- 几种计价方法的区别：

  `实时移动加权平均法`：单据审核时即核算出库成本，月末可通过`材料出库核算`功能重新核算。日常审核时，按单据的审核时间先后，以每一笔单据为独立的核算期间，以上一笔单据结束后的数量和金额为期初，以本笔单据为当期发生，以本笔单据结束后的数量和金额为期末，核算出库单的实际成本。因此同一月的同一物料出库，因中间入库成本的影响，其出库成本可能不一致。

  `移动加权平均法`：和实时移动加权平均法的核算逻辑类似，不只过日常单据审核时不立即核算出库成本，通过`材料出库核算`功能核算成本，也以每一笔单据为独立的核算期间。

  `加权平均法`：日常单据审核时不立即核算出库成本，通过`材料出库核算`功能核算成本，但以每月同组别所有单据汇总核算。因此一般来说，同一月的同一物料，其出库成本应当一致（可能因四舍五入有微差）。

:::

请注意：一个单据，只能生成一个凭证；若干单据可以合并生成一个凭证，不能一个单据生成若干凭证。

#### 3.3.5 内部领用耗用/内部盘亏等-其他出库单

其他出库的主要业务场景：

- 门店盘点过程中，盘亏的物料，可用其他出库单。特别强调，所有的盘亏，必须先核对原因，查找问题责任人，并经部门经理、财务、店总审批，确认为盘亏的，方可通过其他出库单记录。如因相关业务经手人导致的物料损失，应当由其承担一定的经济责任。
- 市场活动物料的耗用，可用领料出库单或其他出库单。
- 门店内部通过库存管理的工具、辅料的，可用领料出库单或其他出库单。
- 油漆，因日常使用时不好计量，月末倒挤的方式核算实际消耗的数量，且不通过维修工单记录并出库时，可用领料出库单或其他出库单。
- 因索赔维修、理赔维修，前述已通过其他入库单记录了换下的物料明细及数量，现应厂商/保险公司要求，交付给厂家/保险公司、销毁等，通过此单据进行记录。

::: note 凭证参考-其他出库单

**其他出库单生成凭证**时，不管正数出库，还是负数出库，由其他出库单【生成凭证】，其凭证格式为：

- 借：**需手工选择**，如：销售费用/管理费用/主营成本/其他业务成本等，该科目具体为哪个科目，根据实际承载的业务、本部财务核算要求、国家财税政策等，金额取对应其他出库单上的实际成本
- 贷：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应其他出库单上的实际成本`**
- 实际成本来源，参见前述销售出库单中的描述
- 其他出库单的业务日期与凭证的记账日期必须在同一期间

:::

请注意：一个单据，只能生成一个凭证；若干单据可以合并生成一个凭证，不能一个单据生成若干凭证。

- 借：**需手工选择**，如：销售费用/管理费用/主营成本/其他业务成本等，该科目具体为哪个科目，根据实际承载的业务、本部财务核算要求、国家财税政策等，金额取对应其他出库单上的实际成本

#### 3.3.7 调整日常出库成本异常-出库成本调整单（业务类型为出库）

出库成本调整单的主要业务场景：

- 当日常出库单出库后，发现出库单的实际成本不正确，但无法修改原出库单时，通过本成本调整单，以错误与正确的差异，调整出库成本。
- 正数，表示原出库单少出了成本，通过此单据增加成本。负数，表示原出库单多出了成本，通过此单据减少成本。
- 当没有结存数量时，采购入库核销有差异时，系统在自动生成入库成本调整单的同时，也会自动生成出库成本调整单，以便结转成本。

::: note 凭证参考-出库成本调整单

**出库成本调整单生成凭证**时，不管正数调整金额，还是负数调整金额，由出库成本调整单【生成凭证】，其凭证格式为：

- 借：`1003 销售成本科目`，如：精品成本/配件成本，金额取对应出库成本调整单上的调整金额，因出库成本调整单金额一般都较小，同时也无法区分是因为调整的何种业务的出库，因此这里统一都取此记账要素关联的科目
- 贷：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应出库成本调整单上的调整金额`**
- 以差异作为调整金额
- 出库成本调整单的业务日期与凭证的记账日期必须在同一期间

:::

#### 3.3.8 公司内部不同仓库/库位的存放位置挪移，移入的为调拨入库单/移出的为调拨出库单

精品配件在不同仓库及（或）库位，发生存放位置的变化时，移入的业务，以调拨入库单为承载。移出的业务，以调拨出库单为承载。必须先调拨出库，再调拨入库，且必须同步有调拨出库和调拨入库，且相同数量、相同库存类型。

库存调拨业务，以库存调拨单为起始单据，通过`直接调拨`功能（需要挪移的物料，其库存类型为普通），自动生成调拨出库单、调拨入库单；或通过关联生成功能（需要挪移的物料，其库存类型为其它），先生成调拨出库、再生成调拨入库。库存调拨单，业务做完后，单据状态必须为`关闭`。

::: danger 特别提醒

- 当要挪移的物料，其库存类型为其它时，请务必在库存调拨单的分录的库存类型字段选择`其它`，并通过关联生成功能，生成调拨出库单、调拨入库单。
- 调拨出库单和调拨入库单，事务类型必须对应匹配，即调拨出库单为`027普通调拨出库`，则调拨入库必须为`028 普通调拨入库`；调拨出库单为`4333-1 调拨出库（库存类型为其它）`，则调拨入库必须为`4333-2 调拨入库（库存类型为其它）`。
- 库存调拨单，作为调拨出库单、调拨入库单的共同上游单据，其业务类型必须为`331 跨仓库调拨（成本价）`，如选择其他将导致成本核算、系统对账、系统结账等功能，均出现问题，请切记！！！
- 调拨出库，不得单独手工新增；调拨入库单，不得单独手工新增；这两者均必须由库存调拨单生成。
- 调拨入库单的物料、数量、库存类型，必须与调拨出库单，一致。表现特征为，库存调拨单的单据状态必须为`关闭`，**禁止任何非关闭状态的库存调拨单存在**。

:::

::: note 凭证参考-调拨出库单/调拨入库单

- **调拨出库单、调拨入库单，不需要生成凭证**。如结账时提示需要生成凭证，则要么在结账时，选项没操作好；要么没遵照上述特别提醒中的事项要求。

:::

### 3.4 库存业务对账主要内容

**日常发生**：

通过上述发生过程，我们可以很容易发现并得到如下恒等关系，本部也是基于这些业务恒等逻辑，要求各店进行业务数据和财务数据的核对。

::: note 库存对账-日常对账

- 入库单据对应的库存业务，以库存商品科目的借方进行记录

- 出库单据对应的库存业务，以库存商品科目的贷方进行记录

- **科目`库存商品-精品/配件`的`借方发生额`合计**

  **= 同期`采购入库单`的`实际成本`**

  **+ 同期`入库成本调整单`的`调整金额`**

  **+ 同期`其他入库单`的`实际成本`**，

  因调拨入库无需生成凭证，业务上不加此单据

- **科目`库存商品-精品/配件`的`贷方发生额`合计**

  **= 同期`销售出库单`的`实际成本`**

  **+ 同期`其他出库单`的`实际成本`**

  **+ 同期`领料出库单`的`实际成本`**

  **+ 同期`出库成本调整单`的`调整金额`**，

  因调拨出库无需生成凭证，业务上不加此单据

:::

**期末余额**：

通过库存业务流转的变化过程，我们可以很容易得到如下恒等关系，本部也是基于这些业务恒等逻辑，要求各店进行业务数据和财务数据的核对。

::: note 库存对账-期末对账

- **科目`库存商品-精品/配件`的`期末余额`合计 = 同期`库存系统期末的结存金额`合计**

- 库存系统期末的结存金额，可通过如下报表查询：

  1. **库存类型收发汇总表**，日期至为`期末最后一天`，单据状态为`审核`，同组织的所有物料的`期末结存金额`；

     路径：`〖精品配件管理〗-〖库存管理〗-〖库存报表〗-〖库存类型收发汇总表〗`

     报表可联查其明细：库存类型收发明细表

  2. **普通物料收发事务汇总表**，日期至为`期末最后一天`，单据状态为`审核`，同组织的所有物料的`期末结存金额`；

     路径：`〖精品配件管理〗-〖库存管理〗-〖库存报表〗-〖普通物料收发事务汇总表〗`

     报表可联查其明细：普通物料收发事务明细表

  3. **存货收发存汇总表**，会计期间`同一期`，单据状态为`审核`，同组织的所有物料的`期末结存金额`；

     路径：`〖财务会计〗-〖存货核算〗-〖报表分析〗-〖存货收发存汇总表〗`

     报表可联查其明细：存货收发存明细表

:::

::: danger 特别提示

- 三个汇总表的主要区别：

  `库存类型收发汇总表`：可以显示库存类型为普通、库存类型为其它、库存类型为赠品等所有库存，并单项汇总显示，**数据最全，一般用此表核对**。

  `普通物料收发事务汇总表`：只汇总显示库存类型为普通的（组别一、组别四）的数据，少了库存类型为其它、库存类型为赠品的数据。

  `存货收发存汇总表`：可以显示是否包含不参与核算的事务类型的数据（组别一、组别四）。汇总的维度不同，如还可以根据记账分类汇总。

- 同时，因按管理要求，组别二、组别三、组别四均要求不得有实际成本，因此这三个报表的金额数据，其期初、当期发生、结存，金额应该一致。但三个报表的数量数据，则因报表的涵盖范围不同，物料明细、数量，会有所不同，请特别注意！！！

- 如果三个报表的结存金额不一致，则说明有不参与核算的事务类型（组别二、组别三、组别四）的单据有实际成本，这不符合管理要求，请尽快纠正。

- 通过**库存类型收发汇总表**，可以查询出库存类型为其它（组别二）、库存类型为赠品（组别三）的实际成本异常的单据。

  通过**库存类型收发汇总表**（只选普通）或**普通物料收发事务汇总表**，与**存货收存发汇总表**对比，可以核查出组别四的异常单据。
  
  存货收存发汇总表**勾选**`显示不参与核算的事务类型`与存货收存发汇总表**不勾选**`显示不参与核算的事务类型`的数据对比，可以核查出哪些物料有异常（对比结存数量、结存金额）。

:::

::: tip 延伸提示：库存账龄分析表的合计怎么与财务余额、库存报表对不上？

- 库存账龄分析表与财务科目余额对相等，前提是财务科目余额与上述三个库存报表对得上。

- 库存账龄分析表的金额合计与库存报表（以库存类型收发汇总表为例）的结存金额对比，如不相符，可能有以下几种原因，供参考：
  - 库存账龄分析法的查询条件不对：不能直接查当期，必须结账后查当期之前的期间：
    1. 库龄分析对象，必须为`期末库存`，不得为即时库存；要统计期末库存，必须存货核算模块结账后才能统计，即要2022年3期的数据，则库存管理必须在2022年4期及以后期间，才能查询。
    2. 是否统计到如下字段，六个字段均不勾选。
    3. 是否统计如下单据，三个单据均不勾选。
    4. 库龄天数，按本部资产财务部统一要求。
  - 库存账龄分析法是以`数量为基准倒查反算`的，因此**当库存报表中显示的结存数据数量为负或零，但结存金额有值，则此部分不会被统计到库存账龄分析表**；同时，这种数量为负或零但有结存金额的数据，实际上也是异常数据，需要处理。
  - 库存报表的截至日期必须为同期间的最后1日，核对结存金额列合计。
  
- 解决办法：
  - 确保查询条件正确；

  - 对库存数量为负的行，请检查该物料的库存类型收发明细表（从上线至当前），核查是哪里出错？为什么会出现负数量？一般物料上设置均是不允许负库存的，如收发明细没有问题但库存结存有问题，可联系本部处理。如果收发明细与库存结存能对得上，但确实发出数量比收入数量+期初数量还多，这就是严重的管理问题了。请自查。

  - 对库存数量为零的行，但有结存金额的，请同样检查该物料的库存类型收发明细表（从上线至当前），检查哪些行成本异常。

    如果是采购入库异常的，应当先做正确的再冲回错误的。

    如果是出库成本错误的，可做出库成本调整单进行调整。

    如果是事务类型错误导致的，应当冲掉错误的事务类型的单据再做正确的事务类型的单据（先冲错误再做正确，还是先做正确再冲错误，这个结合实际库存来，否则可能会提示库存不足）。

- 库存金额异常的常见问题处理思路：（先核查该物料自上线至今`库存类型收发明细表`，再结合采购核销判断采购环节）

  - 采购价格波动较大，导致出库成本核算出错，做出库成本调整单调整多出或少出的成本；如果是非常频繁的或实际根本不是同一物料但混在一起的，应当用个别计价法，并启用批次管理。
  - 采购入库错误，比如数量与成本弄反，请先做正确的采购入库，再冲回错误的采购入库。因错误入库，可能已导致出库成本出错，可结合实际先调整错误的出库成本，再做正确的采购入库，再冲回错误的采购入库。
  - 采购退货与采购入库不一致，如果是制单错误，请先做正确的单据，再冲回错误的。
  - 采购退货与采购入库不一致，如果是折价，则由本门店承担损失，做出库成本调整单来承担。
  - 采购退货与采购入库不一致，只因四舍五入的原因的，如果正负已做采购发票，则只能做出库成本调整单，调整差异。
  - 采购退货与采购入库不一致，只因四舍五入的原因的，但正负入库没有做采购发票，供应商也不会再开发票，则关联生成正负应付单，确保同一物料的应付金额、金额、税额一致，通过核销功能，自动调整。
  - 出库后，因集采降价，可把之前多出的成本，做出库成本调整单，调减出库成本。
  - 物料计价方法不正确，非实时移动加权平均法。如果计价方法是非实时移动加权平均法，则出库单据在审核时，不会立刻核算出库成本，需做`材料出库核算`才会核算出来（单据生成凭证前）。如果又没有做`材料出库核算`，则结存金额会异常，请做出库成本调整单调整。
  - 采购环节都没有问题，入库和出库单据事务类型也正确，但出库成本核算就是出现问题。如果是当期，可把凭证删除，再重新核算出库成本，再重新生成凭证。如果是跨期，则做出库成本调整单调整。
  - 出库时，因除不尽、四舍五入这种客观原因导致的，直接做出库成本调整单调整。
  - 事务类型混合用的，特别是组别一（即最常见的普通的）和组别四（即事务类型含`逐步禁用`字样）混合使用，会导致成本核算错误，进而导致结存金额出错。需要先冲回错误的事务类型的单据，再做正确事务类型的单据。同时，可能还需要结合业务，先做出库成本调整单，把错误的出库成本调整，再冲调单据。

:::

#### 3.5.2 库存金额的核对方法--存货核算的期末对账

该核对方法，是系统自带的功能。

路径：`〖财务会计〗-〖存货核算〗-〖期末处理〗-〖对账〗`

- 期末对账公式：**科目`库存商品-精品/配件`的`期末余额`合计**

  **= 同期`库存系统期末的结存金额`合计**

- 当期借方对账公式：**科目`库存商品-精品/配件`的`借方发生额`合计**

  **= 同期`采购入库单`的`实际成本`**

  **+ 同期`入库成本调整单`的`调整金额`**

  **+ 同期`其他入库单`的`实际成本`**，

  因调拨入库无需生成凭证，业务上不加此单据（勾选：不显示成本价调拨出入库单）

- 当期贷方对账公式：**科目`库存商品-精品/配件`的`贷方发生额`合计**

  **= 同期`销售出库单`的`实际成本`**

  **+ 同期`其他出库单`的`实际成本`**

  **+ 同期`领料出库单`的`实际成本`**

  **+ 同期`出库成本调整单`的`调整金额`**，

  因调拨出库无需生成凭证，业务上不加此单据（勾选：不显示成本价调拨出入库单）

::: note 期末对账简要指引

- 每月结账前，必须对该报表查询三次：

  ①`勾选`：显示不参与核算的事务类型,`勾选`：不显示成本价调拨出入库单

  ②`勾选`：显示不参与核算的事务类型,`不勾选`：不显示成本价调拨出入库单

  ③`不勾选`：显示不参与核算的事务类型,`勾选`：不显示成本价调拨出入库单

- 三次查询结果应当遵循如下原则：

  - ① 和 ② 对比，即是否勾选`不显示成本价调拨出入库单`，**勾选则不包含调拨，不勾选则包含调拨**。

    当本期有调拨业务时，因调拨出库单和调拨入库单均不生成凭证，当勾选时，本期收入金额的库存会包含调拨入库单，这样与本期收入金额的总账就会形成差异、本期发出金额的库存会包含调拨出库单，这样与本期发出金额的总账就会形成差异，但我们已再三强调`调拨出库单和调拨入库单的物料、数量、库存类型必须同步`，因此本期收入金额的库存与总账的差异，会与本期发出金额的库存与总账的差异相等，即两边都同时加上调拨业务的实际成本。

    为此可以得到： ① 和 ② 对比，期末金额对比，结果应该完全一致；本期，要么同时没有差异（没有调拨业务）、要么同时有差异（有调拨业务）。

    **当对比结果，不一致，或差异不相同，则说明调拨业务有问题**，请及时核查库存调拨单、调拨出库单、调拨入库单，并及时纠正。也可通过双击本期收入金额的差额列、本期发出金额的差额列，联查差异明细。

  - ① 和 ③ 对比，即是否勾选`显示不参与核算的事务类型`，**勾选则包含不参与核算的，不勾选则不包含不参与核算的**。

    此选项的含义是指查询的结果是否包含不参与核算的单据数据（即事务类型名称含库存类型为其它、库存类型为赠品、逐步禁用字样的单据），我们已再三强调，`不参与核算的单据，其实际成本字段的值应当为0`，因此勾选与不勾选此项，本期收入金额的库存列、本期发出金额的库存列，均不应该变化。

    为此可以得到： ① 和 ③ 对比，期末金额对比、本期收入金额对比、本期发出金额对比，均应当无差异。

    **当对比结果，不一致，则说明有不参与核算的单据的实际成本字有值**，请及时核查并纠正。也可通过双击本期收入金额的差额列、本期发出金额的差额列，联查差异明细。

- 期末对账的差额列，期初、期末无法双击联查明细。只有本期收入、本期发出，可以双击联查明细。如是因为期初，则需要查询本期之前的期间，一直找到本期收入或本期发出的差额列有值，则可以去联查明细，核查原因。

- 期末对账，可以同时查询连续若干期间的对账结果。

- 存货核算结账和总账结账后，请再次查询该报表，上期期末与本期期初是否一致，如不一致，请反馈给本部。

#### 3.5.3 日常发生的核对

日常发生的核对，按月度（业务日期从当期第1日至当期最后1日）查询已审核的各库存单据，再联查凭证，核对凭证上的数据与单据是否一致，是最基础的操作，是财务人员必需掌握的基础操作，因此本章节就不再赘述。

实际上还可以通过上一小节`存货核算的期末对账`的描述，通过本期收入金额的差异列联查、本期发出金额的差异列联查，可以核查出单据与凭证的差异。

### 3.6 库存业务对账出现差异常见的原因

库存单据生成凭证，这是精品配件核算会计日常记账工作中最基本的一个操作，比较简单只要牢记，入库单据生成凭证在借方；出库单据生成凭证在贷方；不手工凭证，不改金额，不跨期生成凭证，一般来说，不易出错，也不应该出错。

导致库存业务对账出现差异，常见的原因主要有：

#### 3.6.1 物料未绑定记账分类

**常见问题**：

- 物料未绑定记账分类，在生成凭证时，手工直接在凭证上选择了科目。

**解决方案**：

- 在生成凭证时，如果未带出科目，请务必检查单据上的物料的财务资料的记账分类，是否准确关联记账分类。如果没有关联，请批量修改指定后，再生成凭证。

#### 3.6.2 记账要素未关联准确的科目

**常见问题**：

- 单据生成凭证后，带出的科目，与实际预期不相符，比如物料分类为精品的，其记账分类也应当为`精品`，该记账分类中的`1001 物料科目`关联的科目应该为`库存商品-精品`，但错误关联为其它科目。

**解决方案**：

- 请修改对应记账分类中的记账要素绑定的科目。

#### 3.6.3 生成凭证时，修改了带出的科目或修改了物料的记账分类

**常见问题**：

- 比如物料绑定了`精品`，生成凭证时带出`库存商品-精品`，但手工在凭证上改成`库存商品-配件`或其他科目；
- 比如物料绑定了`精品`，生成凭证时带出`库存商品-精品`，但后续又把物料的记账分类绑定了`配件`；
- 以此类推。

**解决方案**：

- 应杜绝对物料反复修改记账分类；
- 生成凭证后，如认为科目不对，请先修改记账分类中的记账要素关联的科目，再生成凭证；
- 生成凭证后，一般应杜绝再修改该物料的记账分类，如确实错了，修改后，请同步通过`存货核算-期末对账`，核查差异明细，同借或同贷调整差异。比如原来是精品，生成凭证也到了精品，但后续发现应该为配件，就把物料记账分类改成了配件，则会导致对账差异，此时，手工凭证，借：库存商品-精品  负，贷：库存商品-配件  正。

#### 3.6.4 入库单据生成凭证时与要求的格式不相符或金额不相等

**常见问题**：

- 入库单据生成凭证时，`库存商品-精品`、`库存商品-配件`应当在借方；如果不在借方，则会导致不平；
- 入库单据生成凭证时，凭证上的金额应当与实际成本/调整金额相等。

**解决方案**：

- 无论如何，应当先按单据生成凭证且方向正确（入库单据在借方、出库单据在贷方）且金额正确（除成本调整单外的单据对应单据上的实际成本，成本调整单对应单据上的调整金额）。与单据相差的，请手工凭证冲回差错。
- 如果单据错了，请先做正确的单据，并生成正确的凭证；再冲回错误的单据并按原错误方式生成凭证。

#### 3.6.5 出库单据生成凭证时与要求的格式不相符或金额不相等

**常见问题**：

- 出库单据生成凭证时，`库存商品-精品`、`库存商品-配件`应当在贷方；如果不在贷方，则会导致不平；
- 出库单据生成凭证时，凭证上的金额应当与实际成本/调整金额相等。

**解决方案**：

- 无论如何，应当先按单据生成凭证且方向正确（入库单据在借方、出库单据在贷方）且金额正确（除成本调整单外的库存单据对应单据上的实际成本，成本调整单对应单据上的调整金额）。与单据相差的，请手工凭证冲回差错。
- 如果单据错了，请先做正确的单据，并生成正确的凭证；再冲回错误的单据并按原错误方式生成凭证。

#### 3.6.6 库存单据的业务归属期间与对应凭证的归属期间不一致

**常见问题**：

- 库存单据的归属期间与对应凭证的归属期间不一致。

**解决方案**：

- 库存单据的归属期间，按其业务日期进行确认，按自然月划分。

- 凭证的归属期间，按记账日期进行确认，按自然月划分。

- 一般这种问题常出现在：

  提前结账，库存单据没按要求修改业务日期；

  月末出库，内部沟通后，应出在下月，但没出在下月，财务把凭证记在下月；

  月初出库，内部沟通后，应出在上月，但没出在上月，财务把凭证记在上月。

- 该问题，会导致对账不平，但万一出现这个，如不好修改时，可不用修改。

- 但类似如下场景必须修改：

  精品销售单或维修业务，在上月末已确认收入，但销售出库单的出库日期却在下月初，此时会导致成本结转错误，必须纠正。

  原则：销售出库单的业务日期，必须小于等于应收单的单据日期；应收单的业务日期必须和单据日期保持一致。

#### 3.6.7 手工凭证

**常见问题**：

- 手工编制了涉及`库存商品-精品`、`库存商品-配件`等需要参与对账的凭证。

视为手工凭证的单据：

- 凭证无上游单据，凭证中包含`库存商品-精品`、`库存商品-配件`等需要参与对账的科目，则一定为手工凭证。
- 凭证有上游单据，凭证中包含`库存商品-精品`、`库存商品-配件`等需要参与对账的科目，但上游单据不为库存单据。
- 凭证有上游单据，凭证中包含`库存商品-精品`、`库存商品-配件`等需要参与对账的科目，且上游单据为库存单据，但手工增加了分录且该分录有`库存商品-精品`、`库存商品-配件`等需要参与对账的科目。

**解决方案**：

- 冲回错误的单据及对应的凭证，再做正确单据及凭证；
- 冲回错误的分录，再做正确的凭证；如错误的本就是多余的，冲回即可。
- 冲回单据与凭证时，应当在分录摘要显著标记为冲回哪期哪号凭证。

## 4.销售发出业务发生过程和对账主要内容

精品配件销售业务的实物交付，需要确认为成本，并需要在业务完结后确认收入，而确认收入的时点与实物出库的时点，并不完全同步，因此为适应此业务特点：

**财务记账**：

- 财务记账时以科目`发出商品-精品配件销售`记录库存商品转入主营业务成本的中间过程。

  借方，表示实物已出库但未确认成本。

  贷方，表示已计入发出商品的，在确认收入后结转成本。

**精品销售业务**：

- 以每月的销售出库单（核心单据类型：精品配件销售单）的实际成本，为当期转入`发出商品-精品配件`的金额；**按月合计，每月`销售出库单（核心单据类型：精品配件销售单）的实际成本`合计，应当等于财务`发出商品-精品配件`科目的借方金额合计**。
- 以每月精品配件销售单关联生成应收单而确认收入（含零结）的时点（应收单的单据日期）为结转成本的时点，辅以《精品配件销售明细表（已结算）》报表，核查每日、连续若干日、每月等时间范围段的数据，而确认相应的收入、成本。**按月合计，`《精品配件销售明细表（已结算）》`报表`本期出库成本`合计应当等于财务`发出商品-精品配件`科目的贷方金额合计**。
- 以《精品配件销售明细表（未结算或未收款）》报表，核查指定时点（一般为月末最后一日）还未确认收入的已出库未结转成本的明细，即精品配件销售发出未结；**如果核查为月末最后一日，`《精品配件销售明细表（未结算或未收款）》`报表`未结转出库成本`合计，应当等于财务`发出商品-精品配件`科目的期末余额**。如不相等，视为存货账账不一致。

其他的各出库，如领料出库单、其他出库单，因为是内部业务，所以在出库时，即可完成成本/费用/支出的确认，无等待确认收入的过程，因此这此业务不属于销售/维修环节，不包含在本节中。

### 4.1 精品配件销售业务发生过程

销售出库按业务特点，区分为精品配件销售、维修业务，通过`核心单据类型`进行体现。

核心单据类型为`精品配件销售单`的，即本项业务为销售业务，由精品配件销售单生成；

核心单据类型为`维修工单`的，即本项业务为维修业务，由维修工单关联生成，参见下述维修发出环节描述。

#### 4.1.1 销售交付实物-销售出库单（核心单据类型为精品配件销售单）

当客户进店购买精品配件，向客户交付实物，或因各种原因客户退还实物，业务上以`销售出库单`为承载记录实物交付过程，即由`精品配件销售单关联生成销售出库单`。

财务记账时以科目`库存商品-精品`，`库存商品-配件`贷方记录实物交付过程。

同时，不考虑确认收入的时点，对方科目全部计入发出，即以`发出商品-精品配件`借方记录实物暂转发出的过程。

::: note 凭证参考--销售出库单（精品配件销售出库）

**销售出库单生成凭证**时，不区分是正数出库还是负数退货，但区分其来源，来源于精品配件销售单的，由核心单据类型为`精品销售单`的销售出库单【生成凭证】，其凭证格式为：

- 借：统一指定为`1406.02 发出商品-精品配件销售`，金额取对应销售出库单上的实际成本
- 贷：`1001 物料科目`，如：`库存商品-精品/配件`，金额取对应销售出库单上的实际成本
- 销售出库单的业务日期与凭证的记账日期必须在同一期间
- 注意，在前述库存环节也讲到了销售出库单生成凭证，这个可不是重复生成凭证，库存环节侧重描述贷方库存商品科目，销售或维修环节侧重描述发出商品科目

:::

::: danger 特别提醒

- 为便于核对当期发生，不管实际业务在发生过程中当时就收款出库并确认收入，还是收入和成本不同步发生，销售出库单生成凭证时，**借方均计入发出商品-精品配件**。禁止：当一起确认收入和成本的就不过发出而直接结转成本，当不一起确认收入和成本时才过发出商品。这会给月末核对时增加难度，万一月末没对平，就要多费很多时间进行查找 。

:::

#### 4.1.2 销售结转成本-查询确认收入的对应出库成本

每一笔精品配件销售单要确认收入，必须该笔单据的所有分录中的`未出库数量为零`，且表头的`是否全部出库为勾选`，才可以关联生成应收单，确认收入。

**精品配件销售单的数量关系**：

精品配件销售单的数量字段，遵循如下恒等关系：

- 每行的`已出库数量`  =  下游`销售出库单同源单分录的数量合计`
- 每行的`未出库数量`  =  同行 `数量  -  退货数量  -  已出库数量`
- 退货数量，根据工具栏中的【退货】，触发可输入，【退货确认】确认输入的结果，会自动触发未出库数量发生变化
- 表头`是否全部出库`，当分录中的`未出库数量`均为0时，该字段应自动勾选。

如在实际操作过程中，发现未遵循如上关系，请检查：

- 精品配件销售单的单据状态是否为`已审核`，如是`变更中`，请对该单据进行【保存】，【提交】，【审核】。
- 请确认【退货】操作后，是否继续进行了【退货确认】。
- 请确认【换货】操作后，是否继续进行了【换货确认】。
- 下游的销售出库单据是否为已审核，如未审核，请审核出库单，多余的单据，请删除。
- 如上操作均没有问题后，但数量还是有问题，请打开此精品销售单，单击工具栏中的【数量反写重算】功能，进行数量的重算。

**应收单更新销售出库单的开票日期逻辑**：

不管是确认正数的收入，还是因退货确认负数的收入，还是零结，均需要精品配件销售单已全部出库后，才可以生成应收单。即该正数出库的就关联生成正数出库，该退回仓库的就关联生成负数出库。

核心逻辑：精品配件销售单，关联生成应收单时，应收单审核后，会把`应收单的单据日期`更新到对应`销售出库单上的开票日期`字段（销售出库单必须审核且开票日期字段必须为空，否则不会被重复更新），并有如下机制：

- 开票日期，是确认收入成本匹配的核心条件之一。本小节描述的结转成本就是依据销售出库单的开票日期来提取的。

- **应收单审核时，只会更新在应收单审核前，对应的销售出库单已审核，且开票日期为空的单据。销售出库单未审核，或开票日期不为空，不会被更新**。

- **`销售出库单，如在应收单审核之后，才审核，则销售出库单上的开票日期，不会被更新`**。门店常在这出现错误，导致看着“明明”已开票已出库，但还挂在未结里，就是因为这里的原因。

- 常见的操作场景有：

  - 正数全部出库审核后，则关联生成正数的应收单，应收单审核后更新这些正数的销售出库单；

  - 后续又可能对其中的部分行或全部行进行退货退库，则关联生成负数的应收单；负数应收单，只更新这些退库的销售出库单，因为正数的销售出库单已被正数的应收单更新了；

  - 正数全部出库审核后，还没有开票推应收单，因业务需要进行退货退库了，则此时关联生成应收单，并审核后，会对正数出库和负数出库都更新开票日期；无需拆分不同的应收单，更无需重复生成应收单。

  - 特别需要提醒的流程：

    如赠送业务，正数出库后，关联生成了正数的零结应收单，完结了业务。后续因各种原因，又需要退货退库，则退库后，仍需要再关联生成一个零结的应收单，否则退库的销售出库无法结转成本，也会在`精品配件销售明细表（未结算或未收款）`中体现出来。

    再比如不管是否是赠送业务，正数全部出库后，还没有生成应收单，因业务需要又对全部已出库进行退货退库，则此时仅需要关联生成一个零结的应收单即可。

    同时还有一种特别场景，精品配件销售单，没有任何出库的情况下就进行了全部退货，此时不需要正数出库也不需要负数退库，但仍需要关联生成一个零结的应收单，否则这个精品配件销售单，就会在`精品配件销售明细表（未结算或未收款）`中体现出来。

  - 不管是什么场景多少金额，该笔精品配件销售单的最后流程应该为应收单。只要最后是应收单，就不需要后面再反复生成重复的多余的应收单。但只要最后不是应收单，必须后面关联生成应收单，完结业务。否则就会在`精品配件销售明细表（未结算或未收款）`中体现出来。

::: warning 提醒

- 由于实际业务发生过程中，可能提前收款并同时开票，如这些预存款开票需要用在精品配件销售业务中， 同样遵循上述机制，流程先后顺序均一样，没有区别。仅是在生成应收单时，转换规则不同。原收款没有同步开票的金额部分，则用普通的规则，当前开票并确认收入；如原已预存款开票的金额部分，则精品配件销售单生成应收单时，其规则请选择`精品销售-应收单（使用预存款开票）`，凭证冲预存款开票确认为实际的收入。

:::

本小节不描述结转收入的内容，相关内容可参见精品配件销售流程。本小节仅描述结转成本的凭证。

从发出商品-精品配件，结转到成本，为手工凭证，其金额按指引的筛选条件查询得出。

::: note 凭证参考--手工凭证

按下述对账查询参考指引，筛选销售出库单上的单据，其凭证格式为：

- 借：`维修及配件成本-配件成本`/`精品成本`，金额按筛选结果中的实际成本列合计
- 贷：统一指定为`1406.02 发出商品-精品配件销售`，金额按筛选结果中的实际成本列合计

:::

### 4.2 销售发出的对账主要内容

**日常发生**：

通过上述发生过程，我们很容易得到如下恒等关系，本部也是基于这些业务恒等逻辑，要求各店进行业务数据和财务数据的核对。

::: note 销售发出--日常对账

- 科目`发出商品-精品配件`的`借方发生额` 的合计  =  销售出库单`核心单据类型`为`精品配件销售单`  且  `业务日期`为`同期`  且  `单据状态`为`审核`的`实际成本`的所有合计
- 科目`发出商品-精品配件`的`贷方发生额` 的合计  =  销售出库单`核心单据类型`为`精品配件销售单`  且  `开票日期`为`同期`  且  `业务日期小于等于查询期间的最后1日`  且  `单据状态`为`审核`的`实际成本`的所有合计

:::

**期末余额**：

通过业务的流程过程、销售出库单的业务日期、销售出库单的开票日期，我们可以还原出月末时点的精品配件销售未结转成本，因此还可以得到如下恒等关系，本部也是基于这些业务恒等逻辑，要求各店进行业务数据和财务数据的核对。

::: note 销售发出--期末对账

- 科目`发出商品-精品配件`的`期末余额`   

  = ① （销售出库单`核心单据类型`为`精品配件销售单`  且  `开票日期`为`空`  且  `业务日期小于等于查询期间的最后1日`  且  `单据状态`为`审核`的`实际成本`  ）

  \+  ②（销售出库单`核心单据类型`为`精品配件销售单`  且  `开票日期`大于等于`查询期间的下期第1日`  且  `业务日期小于等于查询期间的最后1日`  且  `单据状态`为`审核`的`实际成本`  ）

  =  查询日期为查询期间最后1日的《精品配件销售明细表（未结算或未收款）》的未结转出库成本合计（注意是页签为`未结算`的报表）

- ①，即为截至查询期间最后1日，仍没有确认收入的。

  **这里要特别注意，由于历史不规范的问题，可能实际上结了，但开票日期仍为空，导致多出了实际已结但系统未结的单据，这部分销售出库单的开票日期需要处理，比如处理成开票日期与出库日期相等，视同出库时即确认成本**。

- ②，即为截至当前已确认收入，但还原至查询期间最后1日，实际并未确认收入的。

:::

### 4.3 销售发出业务对账查询参考指引

以下查询指引，以截至2022年2月28日的数据为例。

### 4.4 销售发出业务对账出现差异常见的原因

#### 4.4.1 用倒挤方式进行记账

用倒挤方式进行记账，要么本期发生对不上，要么期末余额对不上。如果只管期末余额，倒挤发生额，必然导致本期发生对不上，如果又不去检查发生的差异，就掩盖了问题。

因此，禁止使用倒挤的方式记账。

本期借方发生、贷方发生、期末余额，业务和财务都必须核对一致。

如不一致，需要核查差异。

#### 4.4.2 出库日期在确认收入的应收单的单据日期之后

这是最容易导致对账不平的原因之一。

当门店提前结账时，业务人员在操作销售出库单时，需要修改业务日期，此时可能没注意，修改到了比较远的后面的日期。

比如，1月29日实际出的，但因为提前结账，需要把此笔业务记到2月，结果业务人员修改时，误把业务日期改到了3月1日，甚至更后。然后2月这笔业务确认收入后，就会导致结转成本结转不到这笔，因为此出库出在了3月，最后导致对账不平。

解决方案：

- 方案一：反审核销售出库单，把业务日期改回到正确的2月1日，反审核对应的应收单，并重新审核下。
- 方案二：在2月原结转的成本的基础上增加这笔金额的结转。

另外，大家可以参考这样查询，如果查询有数据，则为异常，仍以2月结转为例：

条件页签：

- 业务日期从：填查询期间的下期的第1日，即3月1日
- 业务日期至：不填
- 单据状态：审核
- 库存组织：选择需要查询的组织
- 最大返回结果数：全部

自定义页签：

- 源单类型名称：等于 精品配件销售单
- 开票日期：小于等于 填查询期间的最后1日，即2月28日

该示例，查询的是开票日期在本期及之前，但出库日期在本期之后，这与业务的流程和逻辑不一致，所以必然导致对账不平。

#### 4.4.3 应收单审核后销售出库单又反审核删除或修改再审核

这是最容易导致对账不平的原因之一。

这种操作，应当杜绝。

如果确有必要修改或删除销售出库单，请务必通知财务人员，对其应收单反审核再重新审核。如果当前暂不出库，则也应当删除应收单，暂不确认收入。待整个精品配件销售单全部出库后，再关联生成应收单。

#### 4.4.4 精品配件销售单的销售日期在销售出库单的业务日期和应收单的单据日期之后

这是最容易导致对账不平的原因之一。

销售日期都在出库或开票日期之后，显然异常。请修正精品配件销售单的销售日期。

#### 4.4.5 历史业务操作不规范

该范围比较广，需要具体问题具体分析。请有需要的门店，联系本部进行指导。

在此需要特别说明：当本部在指导门店处理历史单据时，会对历史单据进行处理，也会指定特定日期作为规范时点。如后续又对已处理的历史单据、规范时点之前的单据进行了操作，如出库、出退、开票、变更等，可能会导致影响原处理结果，如有影响，请务必联系本部指导处理。

#### 4.4.6 对已锁定未出为数量的单据要求释放

在督导门店过程中，往往发现，较长时间前的精品配件销售单，有一行或多行始终未出库，因此我们一般会对该未出库数量进行锁定，即把未出库数量改为0，不再允许出库，并要求门店对该单据确认收入。但在后续的过程中，客户确又来店，要求提供此精品配件，此时，就需要还原未出库数量，业务人员把物料出给客户，实际上此时需要再推一个零结的应收单（不管该单总价值是否为0，因为在锁定未出库时已完成全部收入的确认，不能再确认有金额的收入，通过零结的应收单，进而标记重新出库的物料应该结转成本的时点）以结转成本。但门店可能没有操作，就可能导致对账不平。

#### 4.4.7 系统反写错误

分两种情况：

一是真的反写错误，比如已出库数量不等于下游出库单的数量、应收单审核确实在销售出库单之后，但没有更新出库单的开票日期，或开票日期与实际不符；这类问题，需要联系本部处理。

二是假的反写错误，看似已出完库、已开完票，结转成本也做完了。这时业务人员又反审核了销售出库单，重新出库或修改之类的操作，再重新审核，这时销售出库单的审核时间在应收单的审核时间之后，导致不会被更新销售出库单的开票状态、开票日期，进而导致对账不平。

## 5.维修发出业务发生过程和对账主要内容

维修业务的实物交付，需要确认为成本，并需要在业务完结后确认收入，而确认收入的时点与实物出库的时点，并不完全同步，因此为适应此业务特点：

**财务记账**：

- 财务记账时以科目`发出商品-维修`记录库存商品转入主营业务成本的中间过程。

  借方，表示实物已出库但未确认成本。

  贷方，表示已计入发出商品的，在确认收入后结转成本。

**维修业务**：

- 以每月的销售出库单（核心单据类型：维修工单）的实际成本，为当期转入`发出商品-维修`的金额；**按月合计，每月`销售出库单（核心单据类型：维修工单）的实际成本`合计，应当等于财务`发出商品-维修`科目的借方金额合计**。
- 以每月维修结算单关联生成应收单而确认收入（含零结）的时点（应收单的单据日期）为结转成本的时点，辅以《维修明细表（已结算）》报表，核查每日、连续若干日、每月等时间范围段的数据，而确认相应的收入、成本。**按月合计，`《维修明细表（已结算）》`报表`配件本期出库成本`合计应当等于财务`发出商品-维修`科目的贷方金额合计**。
- 以《维修明细表（未结算）》报表，核查指定时点（一般为月末最后一日）还未确认收入的已出库未结转成本的明细，即维修未结；**如果核查为月末最后一日，`《维修明细表（未结算）》`报表`未结转出库成本`合计，应当等于财务`发出商品-维修`科目的期末余额**。如不相等，视为存货账账不一致。

其他的各出库，如领料出库单、其他出库单，因为是内部业务，所以在出库时，即可完成成本/费用/支出的确认，无等待确认收入的过程，因此这此业务不属于销售/维修环节，不包含在本节中。

### 5.1 维修业务发生过程

按业务特点，区分为精品配件销售、维修业务，通过`核心单据类型`进行体现。

核心单据类型为`精品配件销售单`的，即该项业务为销售业务，由精品配件销售单生成；参见上述销售发出章节描述。

核心单据类型为`维修工单`的，即该项业务为维修业务，由维修工单关联生成。

#### 5.1.1 维修交付实物-销售出库单（核心单据类型为维修工单）

当客户进店维修车辆，车间技师向库房领用并用于维修作业，或因各种原因从维修作业退还实物，业务上以`销售出库单`为承载记录实物交付过程，即由`维修工单关联生成销售出库单`。

财务记账时以科目`库存商品-精品`，`库存商品-配件`贷方记录实物交付过程。

同时，不考虑确认收入的时点，对方科目全部计入发出，即以`发出商品-维修`借方记录实物暂转发出的过程。

::: note 凭证参考--销售出库单（维修出库）

**销售出库单生成凭证**时，不区分是正数出库还是负数退货，但区分其来源，来源于维修工单的，由核心单据类型为`维修工单`的销售出库单【生成凭证】，其凭证格式为：

- 借：统一指定为`1406.01 发出商品-维修`，金额取对应销售出库单上的实际成本
- 贷：`1001 物料科目`，如：**`库存商品-精品/配件，金额取对应销售出库单上的实际成本`**
- 销售出库单的业务日期与凭证的记账日期必须在同一期间
- 注意，在前述库存环节也讲到了销售出库单生成凭证，这个可不是重复生成凭证，库存环节侧重描述贷方库存商品科目，销售或维修环节侧重描述发出商品科目

:::

::: danger 特别提醒

- 为便于核对当期发生，不管是一般维修保养当天就完成作业的，还是诸如理赔、索赔等需要较长时间结算的，销售出库单生成凭证时，**借方均计入发出商品-维修**，禁止：当一起确认收入和成本的就不过发出而直接结转成本，当不一起确认收入和成本时才过发出商品。这会给月末核对时增加难度，万一月末没对平，就要多费很多时间进行查找 。

:::

#### 5.1.2 维修结转成本-查询确认收入的对应出库成本

每一笔维修业务，在维修作业完成后，需要进行维修结算。

维修结算以维修类型对应的付费类别中对应的结算对象为区分，主要有四大类流程及财务处理：

##### 5.1.2.1 厂家保修业务

- **厂家保修业务：结算对象为厂家**，维修工单上维修类型主要有`保修索赔`，`定期保养（厂家结算）`，`厂家首保`等；这类业务，需要按`索赔完整流程`流转业务。由索赔单关联生成的维修结算单（简称索赔结算单），生成应收单确认收入。同时，应收单审核后，会把应收单的单据日期更新到对应的销售出库单的开票日期字段。

  这类业务对应的销售出库单，其事务类型主要为：

  - 2520 维修出库（索赔）
  - 2521 维修退库（索赔）

  也即，在结转索赔维修业务的成本时，主要对应这些事务类型的销售出库单。

本小节不描述结转收入的内容，相关内容可参见索赔维修流程。本小节仅描述结转成本的凭证。

从发出商品-维修，结转到成本，为手工凭证，其金额按指引的筛选条件查询得出。

- **钣喷自费业务：结算对象为客户**，维修工单上维修类型主要有`保险事故（垫付）`，`钣喷自费`等；这类业务，需要按`一般维修完整流程`流转业务。由维修工单关联生成的维修结算单（简称工单结算单），生成应收单确认收入。同时，应收单审核后，会把应收单的单据日期更新到对应的销售出库单的开票日期字段。

### 5.4 维修发出业务对账出现差异常见的原因

#### 5.4.1 用倒挤方式进行记账

用倒挤方式进行记账，要么本期发生对不上，要么期末余额对不上。如果只管期末余额，倒挤发生额，必然导致本期发生对不上，如果又不去检查发生的差异，就掩盖了问题。

因此，禁止使用倒挤的方式记账。

本期借方发生、贷方发生、期末余额，业务和财务都必须核对一致。

如不一致，需要核查差异。

#### 5.4.2 出库日期在确认收入的应收单的单据日期之后

这是最容易导致对账不平的原因之一。

当门店提前结账时，业务人员在操作销售出库单时，需要修改业务日期，此时可能没注意，修改到了比较远的后面的日期。

比如，1月29日实际出的，但因为提前结账，需要把此笔业务记到2月，结果业务人员修改时，误把业务日期改到了3月1日，甚至更后。然后2月这笔业务确认收入后，就会导致结转成本结转不到这笔，因为此出库出在了3月，最后导致对账不平。

解决方案：

- 删除维修工单下游单据，反审核销售出库单，把业务日期改回到正确的2月1日，再重新维修工单的下游各单单据。

另外，大家可以参考这样查询，如果查询有数据，则为异常，仍以2月结转为例：

条件页签：

- 业务日期从：填查询期间的下期的第1日，即3月1日
- 业务日期至：不填
- 单据状态：审核
- 库存组织：选择需要查询的组织
- 最大返回结果数：全部

自定义页签：

- 源单类型名称：等于 维修工单
- 开票日期：小于等于 填查询期间的最后1日，即2月28日

该示例，查询的是开票日期在本期及之前，但出库日期在本期之后，这与业务的流程和逻辑不一致，所以必然导致对账不平。

该示例，并不能查询开票日期和出库日期在同一期间，但开票日期比出库日期要小的数据。如果存在此类数据，按月查询，可能没有什么影响，但日常按月内时间段查询，就会与实际不相符。

#### 5.4.3 维修工单的进厂时间在销售出库单的业务日期和应收单的单据日期之后

这是最容易导致对账不平的原因之一。

维修工单的进厂时间，在出库或开票日期之后，显然异常。请修正维修工单的进厂时间。

维修工单的进厂时间，不管是否提前结账，均无需修改。只有当门店是以补单的方式录入数据至系统，比如当天补前一天的，则可以修改进厂时间为前一天。

维修工单的进厂时间，是各种维修报表的取值判断的重要条件之一，因此请所有门店在填制维修工单时，非本部指导要求下，均不得修改`进厂时间`。

维修工单异常进厂时间，请参考报表：`《数据检查-维修工单异常进厂时间参考核对表》`（路径：〖维修管理〗-〖维修业务〗-〖维修工单〗-〖数据检查-维修工单异常进厂时间参考核对表〗）。

::: warning 提醒

- 部分门店，在以厂家DMS为主，在EAS系统为辅补入数据。往往会出现第二天补做头一天的业务，这时可能库管人员会注意到销售出库单的业务日期、财务人员会注意到应收单的单据日期，但往往服务顾问会忽略进厂时间的修改，此时，如补头一天的单据，服务顾问应当修改维修工单的进厂时间为头一天。否则可能导致进厂时间比销售出库的时间晚，这与实际不符。也必将导致成本结转出现问题。

:::

#### 5.4.4 历史业务操作不规范

该范围比较广，需要具体问题具体分析。请有需要的门店，联系本部进行指导。

在此需要特别说明：当本部在指导门店处理历史单据时，会对历史单据进行处理，也会指定特定日期作为规范时点。如后续又对已处理的历史单据、规范时点之前的单据进行了操作，可能会导致影响原处理结果，如有影响，请务必联系本部指导处理。

#### 5.4.5 系统反写错误

比如应收单审核，但没有更新出库单的开票日期，或更新的开票日期与实际不符；维修出库与维修工单的关联关系丢失，上查下查提示没有单据；这类问题，需要联系本部处理。

另外，大众体系门店，因为索赔业务的特殊，需要由维修结算单推应付单，这里往往反写了销售出库单的开票日期，当单据的工时费为0时，往往忘记生成零结的应收单，导致销售出库单看似应结转成本，但在维修明细表（已结算）报表中却没有结算。不应由应付单反写开票日期，如遇到此问题，请反馈给本部处理。同时提醒，当工时费为零，配件费不为零时，一定要记得生成一个与应付单单据日期一致的零结的应收单。（考虑到反写、且挂着未收款，拟调整大众索赔开票结算流程。）
